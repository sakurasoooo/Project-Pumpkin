# 程序部门 (Programmer) 开发任务与计划

欢迎来到程序部门工作区。本项目采用 Unity C# 进行开发，本目录将用于存放代码规范、架构图或临时脚本记录。最终的正式代码请放置于根目录的 `Scripts` 文件夹中。

## 🎯 当前目标
实现《倭瓜射手大冒险》的核心游戏循环，搭建可扩展的架构，并保证在无限跑酷模式下的性能稳定（无 GC 掉帧）。

## 📋 任务列表

### 1. 核心架构与管理器 (Priority: High)
- [x] **`GameManager.cs`**：
  - 管理游戏状态机 (Init, Playing, GameOver)。
  - 管理分数统计与难度曲线（控制全局的速度倍率）。
- [x] **`MapScroller.cs`**：
  - 实现草地、泥土、背景层的无缝向左滚动。通过读取 GameManager 的速度倍率来动态改变滚动速度。
  - 采用双图交替补位法实现，具体请参考设计文档：[MapScrolling_Design.md](MapScrolling_Design.md)。

### 2. 角色控制器与射击系统 (Priority: High)
- **`PlayerController.cs`**：
  - 实现持续的小幅度自动跳跃。
  - 监听屏幕点击（或者 InputSystem），实现角色的主动跳跃及二连跳逻辑。
- **`PlayerShooter.cs`**：
  - 实现固定频率的自动向前方射击机制。
  - 编写武器状态机或策略模式，以支持拾取道具后无缝切换为“玉米大炮”或“豌豆机枪”模式。

### 3. 敌人与生成器系统 (Priority: High)
- **`EnemySpawner.cs`**：
  - 在屏幕右侧视口外，根据策划设定的频率动态生成敌人或道具。
  - 必须使用 **对象池 (Object Pool)** 来管理生成的敌人与道具。
- **`Enemy.cs`**：
  - 基础闲置状态。
  - 碰撞检测：与玩家碰撞（触发游戏结束）、与子弹碰撞（触发死亡特效、销毁并在池中回收、通知 GameManager 加分）。

### 4. 性能与表现优化 (Priority: Medium)
- 引入 `Feel` / `DOTween` 插件，在角色跳跃、武器开火以及敌人受击时添加缓动动画（形变缩放）和屏幕震动效果。
- 确保所有的子弹生成同样使用对象池，消除因为高频射击带来的内存分配。

## 🗓️ 交付与反馈
- **里程碑1**：完成可以无尽循环的灰盒跑酷场景，包含基础的跳跃和射击。
- **里程碑2**：接入美术资源，完善武器切换和性能优化。最后确保打包 APK 在 Android 设备上不出现卡死。
